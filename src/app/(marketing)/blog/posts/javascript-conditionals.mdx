# Introduction to JavaScript Conditionals

Conditional statements are the building blocks of decision-making in programming. They allow your code to make choices and execute different actions based on different conditions. Think of them as the "if-then" logic you use in everyday life: "If it's raining, then I'll take an umbrella."

## The if Statement

The `if` statement is the fundamental building block of conditional logic in JavaScript. It allows your code to make decisions by executing code blocks only when certain conditions are met. Understanding `if` statements is crucial for writing dynamic and responsive code.

### Basic Syntax and Usage

The basic syntax follows this pattern:

<br />

<CodeSnippet code={`if (condition) {
    // code to execute if condition is true
}`} language="javascript" filename="if-statement.js" />

<br />

Let's break down a practical example:

<br />
<CodeSnippet code={`// Example 1: Basic if statement
const temperature = 25;

if (temperature > 30) {
    console.log("It's a hot day!");
}

// Example 2: Using multiple conditions
const isWeekend = true;
const isRaining = false;

if (isWeekend && !isRaining) {
    console.log("Perfect day for outdoor activities!");
}

// Example 3: Checking string values
const userRole = "admin";

if (userRole === "admin") {
    console.log("Welcome to the admin panel");
}`} language="javascript" filename="practical-examples.js" />

<br />

### Common Gotchas and Best Practices

1. **Truthy vs Falsy Values:**

<br />
<CodeSnippet code={`// Be careful with truthy/falsy values
if (0) {
    console.log("This won't run"); // 0 is falsy
}

if ("") {
    console.log("This won't run"); // empty string is falsy
}

if ([]) {
    console.log("This will run!"); // empty array is truthy
}

// Better to be explicit
if (array.length > 0) {
    console.log("Array has items");
}`} language="javascript" filename="truthy-falsy.js" />

<br />

2. **Single Line if Statements:**

<CodeSnippet code={`// While this works:
if (temperature > 30) console.log("Hot!");

// This is better for readability:
if (temperature > 30) {
    console.log("Hot!");
}`} language="javascript" filename="single-line.js" />

<br />

## Using else and else if

The `else` and `else if` statements extend the power of conditionals by allowing you to handle multiple scenarios. They create a decision tree where only one branch will execute.

### Basic Structure

<CodeSnippet code={`// Complex decision tree example
const score = 85;
const isPremiumUser = true;

if (score >= 90) {
    console.log("A Grade");
} else if (score >= 80) {
    console.log("B Grade");
    
    // Nested if statements for more complex logic
    if (isPremiumUser) {
        console.log("Eligible for advanced courses");
    }
} else if (score >= 70) {
    console.log("C Grade");
} else {
    console.log("Need improvement");
    console.log("Contact a tutor for help");
}

// Real-world example: User authentication
const user = {
    isLoggedIn: true,
    role: "editor",
    subscription: "pro"
};

if (!user.isLoggedIn) {
    console.log("Please log in to continue");
} else if (user.role === "admin") {
    console.log("Welcome to the admin dashboard");
} else if (user.role === "editor" && user.subscription === "pro") {
    console.log("Welcome to the pro editor dashboard");
} else {
    console.log("Welcome to the basic dashboard");
}`} language="javascript" filename="else-if.js" />

### Performance Tips

1. Arrange your conditions from most specific to least specific
2. Use early returns for better performance
3. Consider using switch statements for many conditions
4. Avoid deeply nested if/else statements (consider early returns or switch statements instead)

<br />

## Switch Statements

When you have multiple conditions to check against a single value, the `switch` statement can be cleaner than multiple `if/else` statements:

<CodeSnippet code={`const dayOfWeek = "Monday";

switch (dayOfWeek) {
    case "Monday":
        console.log("Start of the work week!");
        break;
    case "Friday":
        console.log("Weekend is coming!");
        break;
    case "Saturday":
    case "Sunday":
        console.log("It's the weekend!");
        break;
    default:
        console.log("It's a regular day.");
}

// Output: "Start of the work week!"
// The switch statement checks the value of dayOfWeek against each case`} language="javascript" filename="switch.js" />

<br />

## Comparison Operators

JavaScript provides several operators to compare values:

<CodeSnippet code={`// Equal to (==) - compares values
console.log(5 == "5");    // true (converts types)

// Strictly equal to (===) - compares values and types
console.log(5 === "5");   // false (different types)

// Greater than (>)
console.log(10 > 5);      // true

// Less than (<)
console.log(5 < 10);      // true

// Greater than or equal to (>=)
console.log(5 >= 5);      // true

// Less than or equal to (<=)
console.log(4 <= 5);      // true

// Not equal to (!=)
console.log(5 != "6");    // true`} language="javascript" filename="comparison-operators.js" />

<br />

## Logical Operators

Combine multiple conditions using logical operators:

<CodeSnippet code={`// AND operator (&&) - both conditions must be true
const age = 25;
const hasLicense = true;

if (age >= 18 && hasLicense) {
    console.log("You can drive!"); // This will run
}

// OR operator (||) - at least one condition must be true
const isWeekend = false;
const isHoliday = true;

if (isWeekend || isHoliday) {
    console.log("No work today!"); // This will run
}

// NOT operator (!) - reverses a boolean value
const isLoggedIn = false;

if (!isLoggedIn) {
    console.log("Please log in"); // This will run
}`} language="javascript" filename="logical-operators.js" />

## Best Practices

1. **Use Clear Conditions**
   - Write conditions that are easy to understand
   - Break complex conditions into smaller parts
   - Use meaningful variable names

2. **Default Cases**
   - Always include else or default cases
   - Handle unexpected scenarios

3. **Use Strict Equality**
   - Prefer === over == to avoid type coercion
   - Be explicit about your comparisons

4. **Keep it Simple**
   - Avoid deeply nested conditions
   - Consider using early returns
   - Break complex logic into functions
